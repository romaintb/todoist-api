# Repository Guidelines

## Project Structure & Module Organization
- `src/` contains the Rust library code.
- `tests/` contains integration tests.

## Build, Test, and Development Commands
- Build: `cargo build`
- Tests: `cargo test`

## Coding Style & Naming Conventions
- Use `rustfmt` and `clippy`.

## Testing Guidelines
- Prefer integration tests for API coverage.

## Commit & Pull Request Guidelines
- Keep semver in mind for public API changes.

## Configuration Tips
- API tokens are required for integration tests.
